### Discretion Decisions

**Category filter: single-select (not multi-select).** Rationale:
1. Matches the simplicity principle from CONTEXT.md ("filter chips should feel lightweight").
2. Single-select is clearer for the chip label (shows one category name vs "3 categories").
3. Multi-select adds combinatorial complexity to the API and state management for marginal UX benefit.
4. Users who want to see all expenses can simply clear the category filter.

**Category picker: bottom sheet with list (not horizontal chip row or dropdown).** Rationale:
1. Users may have many categories -- a scrollable bottom sheet handles 20+ categories gracefully.
2. Horizontal chip row would require a second tap to scroll to off-screen categories.
3. Bottom sheet matches the date preset picker style for visual consistency.
4. Each category row can show its icon and color for easy identification.

**Date preset calculation: calendar-based (not rolling).** Rationale:
1. "This Week" = Monday through Sunday of the current week. "This Month" = 1st through last day of the current month.
2. Calendar-based is more intuitive for expense tracking ("what did I spend this month?").
3. "Last Month" = 1st through last day of the previous calendar month.
4. "Today" = midnight to midnight of the current date.

**Filter state: separate provider (not extending ExpenseNotifier).** Rationale:
1. Filter state is UI state, separate from expense data state.
2. A separate `filterStateProvider` can be watched by `HistoryScreen` to trigger `loadExpenses()` with filter params.
3. Keeps `ExpenseNotifier` focused on CRUD operations.

**Loading indicator when switching filters:** Replace the expense list with a centered `CircularProgressIndicator` while the filtered request is in flight. The filter bar stays pinned and visible during loading so users can see what filters are active.

## Interaction Contracts

### Apply Date Filter

1. User taps the "All Dates" filter chip.
2. Bottom sheet slides up with preset options: Today, This Week, This Month, Last Month, Custom Range.
3. User taps a preset (e.g., "This Month").
4. Bottom sheet closes. Chip label updates to "This Month". Chip shows selected state (checkmark + primary border).
5. Expense list shows `CircularProgressIndicator` briefly while filtered results load.
6. List updates to show only expenses within the selected date range.

### Apply Custom Date Range

1. User taps the "All Dates" filter chip.
2. Bottom sheet slides up with preset options.
3. User taps "Custom Range".
4. Bottom sheet closes. Flutter `showDateRangePicker` opens.
5. User selects start and end dates, taps "Save".
6. Chip label updates to short date range (e.g., "Mar 1 -- Mar 15").
7. List reloads with the custom date range applied.

### Apply Category Filter

1. User taps the "All Categories" filter chip.
2. Bottom sheet slides up with "Clear" at top (if a category is already selected) followed by all user categories.
3. User taps a category (e.g., "Food").
4. Bottom sheet closes. Chip label updates to "Food". Chip shows selected state.
5. List reloads with only expenses in the "Food" category.

### Clear a Single Filter

1. User taps an active filter chip (e.g., chip showing "This Month").
2. The corresponding picker bottom sheet opens.
3. For date: user sees presets with a checkmark on the active one. A "Clear" `ListTile` at the top resets to "All Dates".
4. For category: user taps the "Clear" option at the top of the list.
5. Chip reverts to inactive label. List reloads without that filter.

Alternative clear path: tapping the `FilterChip` `onDeleted` trailing X icon clears the filter directly without opening the picker. Use `FilterChip.onDeleted` for active chips to show a trailing X that clears on tap.

### Combine Filters

1. User applies a date filter (e.g., "This Month").
2. User then applies a category filter (e.g., "Food").
3. Both chips show active state. List shows only "Food" expenses from "This Month".
4. User can clear either filter independently.

### Empty Filter Results

1. Filters are active but no expenses match.
2. Instead of the expense list, show centered empty state: filter-off icon + "No expenses match these filters" + "Try adjusting your date range or category."
3. Filter bar remains visible and interactive so user can modify filters.

### States

| State | Visual |
|-------|--------|
| No filters active | Filter bar shows "All Dates" + "All Categories" (both unselected). Full expense list below. |
| Date filter active | Date chip selected (checkmark, primary border, active label). Category chip unchanged. Filtered list below. |
| Category filter active | Category chip selected (checkmark, primary border, category name). Date chip unchanged. Filtered list below. |
| Both filters active | Both chips selected. Combined-filtered list below. |
| Filters loading | Filter bar pinned. List area shows centered `CircularProgressIndicator`. |
| Filters empty result | Filter bar pinned. List area shows empty filter state (icon + message). |
| Date preset sheet open | Modal bottom sheet with 5 options. Active preset has leading checkmark. |
| Category sheet open | Modal bottom sheet with "Clear" + category list. Selected category has leading checkmark. |
| History loading (initial) | `CircularProgressIndicator` centered (existing, unchanged) |
| History empty (no expenses) | "No expenses yet" centered text (existing, unchanged) |
| History error | Error text centered (existing, unchanged) |

## Filter Bar Layout

```
+------------------------------------------------------------------+
| [16px] [FilterChip: All Dates] [8px] [FilterChip: All Categories] |
+------------------------------------------------------------------+
|                                                                    |
|  Expense list (scrollable)                                        |
|  +------------------------------------------------------------+  |
|  | [CatChip]  $12.34                              Mar 15       |  |
|  |            Lunch with team                                  |  |
|  +------------------------------------------------------------+  |
|  | [CatChip]  $5.00                               Mar 14       |  |
|  |            Coffee                                           |  |
|  +------------------------------------------------------------+  |
|                                                                    |
+------------------------------------------------------------------+
```

Implementation approach:
- `Column` with two children: filter bar (fixed) and `Expanded` list area.
- Filter bar: `Padding(horizontal: 16, vertical: 8)` containing a `Row` of `FilterChip` widgets with `SizedBox(width: 8)` between them.
- List area: existing `ListView.builder` wrapped in `Expanded`.
- Filter bar does NOT scroll with the list -- it stays pinned at the top via `Column` layout.

## Date Preset Bottom Sheet Layout

```
+----------------------------------+
| Filter by Date          [drag]   |
+----------------------------------+
| [x] Clear                        |  <-- only when a preset is active
+----------------------------------+
|     Today                        |
|     This Week                    |
| [check] This Month               |  <-- active preset
|     Last Month                   |
|     Custom Range                 |
+----------------------------------+
```

- `showModalBottomSheet` with `isScrollControlled: false`.
- Title row: `Padding(all: 16)` with `Text('Filter by Date', style: titleMedium)`.
- Each option: `ListTile(leading: Icon(Icons.check) or null, title: Text(label), onTap: ...)`.
- Active preset shows `Icons.check` in leading position with `colorScheme.primary` color.
- "Clear" shows `Icons.close` in leading position. Only visible when a date filter is active.
- "Custom Range" shows `Icons.date_range` in leading position.

## Category Picker Bottom Sheet Layout

```
+----------------------------------+
| Filter by Category      [drag]   |
+----------------------------------+
| [x] Clear                        |  <-- only when a category is selected
+----------------------------------+
| [colored_icon] Food               |
| [colored_icon] Transport          |
| [check] [colored_icon] Shopping   |  <-- active category
| [colored_icon] Entertainment      |
+----------------------------------+
```

- `showModalBottomSheet` with `isScrollControlled: true` and `maxHeight: 0.5` (half screen) for long category lists.
- Title row: `Padding(all: 16)` with `Text('Filter by Category', style: titleMedium)`.
- "Clear" option: `ListTile(leading: Icon(Icons.close), title: Text('Clear'))`. Only visible when a category is selected.
- Each category: `ListTile` with `leading: _CategoryChipSmall(category)` (reuse existing 36x36 circular chip), `title: Text(category.name)`, `trailing: Icon(Icons.check)` if selected.
- Tapping any category closes the sheet and applies the filter.
