## Typography

All values inherit from Material 3 `TextTheme` generated by `ColorScheme.fromSeed`. No custom fonts or overrides.

| Role | TextTheme Style | Size | Weight | Line Height |
|------|----------------|------|--------|-------------|
| Body | bodyMedium | 14px | 400 (regular) | 1.43 |
| Label | bodySmall | 12px | 400 (regular) | 1.33 |
| Heading | titleMedium | 16px | 500 (medium) | 1.5 |
| Display | headlineSmall | 24px | 400 (regular) | 1.33 |

Phase-specific usage:
- Expense row amount: `titleMedium` (16px, weight 500)
- Expense row date / secondary text: `bodySmall` (12px, weight 400)
- Expense row note: `bodyMedium` (14px, weight 400)
- AppBar title ("Edit Expense"): AppBar default (titleLarge, 22px, weight 400)

- Confirmation dialog body: AlertDialog default (bodyMedium, 14px)

Source: Existing `_ExpenseTile` uses `theme.textTheme.titleMedium` for amount, `theme.textTheme.bodySmall` for date.

## Color

All colors derive from `ColorScheme.fromSeed(seedColor: Colors.teal)`. No custom hex values; reference semantic color roles.

| Role | Semantic Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `colorScheme.surface` | Scaffold background, list background |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | Card surfaces (not used this phase -- flat list) |
| Accent (10%) | `colorScheme.primary` (teal-derived) | Save button fill, AppBar text, active states |
| Destructive | `Colors.red` (literal) | Swipe-to-delete background, delete button text, delete dialog action |
| Category colors | Per-category hex from server | CategoryChip background (15% opacity) and icon/text (full opacity) |

Accent reserved for: Save button (`FilledButton`), AppBar title color, date picker active date.

Destructive reserved for: Swipe-to-delete background, delete confirmation "Delete" button text, delete icon in swipe background, delete icon in AppBar (if placed there).

Source: Existing `categories_screen.dart` uses `Colors.red` for Dismissible background and `Icons.delete` with white. Existing `expense_form_screen.dart` uses `FilledButton` (primary color) for save.

### Discretion Decision: Delete Button Placement

Place delete as a **trash icon in the AppBar** (not a red button at bottom). Rationale:
1. Matches Material 3 convention for destructive secondary actions on detail/edit screens.
2. Keeps the bottom of the form clean for the primary "Save" action.
3. Avoids accidental taps near the save button.
4. Consistent with how many Material apps handle edit + delete on the same screen.

## Interaction Contracts

### Tap to Edit

1. User taps an expense row in the History tab.
2. Navigation pushes `/expenses/:id/edit` route.
3. `ExpenseFormScreen` opens with all fields pre-filled from the tapped expense.
4. AppBar title reads "Edit Expense".
5. AppBar trailing action shows trash icon (`Icons.delete`).
6. Amount field does NOT auto-focus in edit mode (user may only want to change one field).
7. User modifies any field(s) and taps "Save".
8. PUT request fires. Button shows `CircularProgressIndicator` (same as create).
9. On success: `Navigator.pop(context)` + SnackBar "Expense updated".
10. On failure: SnackBar "Failed to update expense".

### Swipe to Delete (History List)

1. User swipes an expense row from right to left.
2. Red background with white delete icon reveals behind the row (same as Phase 3 categories).
3. Confirmation dialog appears: "Delete this $12.34 expense?" with Cancel and Delete buttons.
4. If confirmed: row animates out. SnackBar "Expense deleted".
5. If cancelled: row snaps back to original position.

### Delete from Edit Screen

1. User taps trash icon in AppBar while editing an expense.
2. Same confirmation dialog: "Delete this $12.34 expense?" with Cancel and Delete buttons.
3. If confirmed: `Navigator.pop(context)` back to history. SnackBar "Expense deleted".
4. If cancelled: dialog closes, edit screen remains.

### States

| State | Visual |
|-------|--------|
| History loading | `CircularProgressIndicator` centered (existing) |
| History empty | "No expenses yet" centered text (existing) |
| History error | Error message centered text (existing) |
| History loaded | `ListView.builder` with expense rows |
| Edit form saving | Save button shows `CircularProgressIndicator` (same as create) |
| Edit form save error | SnackBar "Failed to update expense" |
| Delete in progress | Dialog dismisses, row animates out via `Dismissible` |
| Delete error | SnackBar "Failed to delete expense" |

## Expense Row Layout (Enhanced _ExpenseTile)

```
+----------------------------------------------------------+
| [CategoryChip]  Amount ($12.34)           Date (Mar 15)  |
|                 Note or date subtitle                     |
+----------------------------------------------------------+
```

Specific layout:
- `ListTile` with `leading: CategoryChip(category: matchedCategory)`
- `title: Text(Expense.formatCents(amountCents))` using `titleMedium`
- `subtitle: Text(note.isNotEmpty ? note : dateStr)` using `bodyMedium`
- `trailing: Text(dateStr)` using `bodySmall` -- always shown for date regardless of subtitle content
- When note is non-empty: subtitle shows note, trailing shows date
- When note is empty: subtitle shows date, trailing shows date (trailing can be omitted to avoid duplication -- Claude's discretion in implementation)
- `onTap: () => context.push('/expenses/${expense.id}/edit', extra: expense)`
