    - Categories table has: id UUID PK, user_id UUID FK to users, name TEXT, icon TEXT, color TEXT, sort_order INTEGER, created_at, updated_at

  <action>
    1. Create `server/internal/db/migrations/00003_categories.sql` with goose Up/Down:
       - categories table: id UUID PK DEFAULT gen_random_uuid(), user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, name TEXT NOT NULL, icon TEXT NOT NULL, color TEXT NOT NULL, sort_order INTEGER NOT NULL DEFAULT 0, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
       - CREATE INDEX idx_categories_user_id ON categories(user_id)
       - NO unique index on (user_id, sort_order) -- per research, gaps are acceptable and unique constraint causes issues during batch reorder
       - Down: DROP TABLE IF EXISTS categories

    2. Create `server/internal/db/queries/categories.sql` with sqlc queries:
       - CreateCategory :one -- INSERT with sort_order = (SELECT COALESCE(MAX(sort_order), -1) + 1 FROM categories WHERE user_id = $1)

       - GetCategoryByID :one -- SELECT * WHERE id AND user_id
       - UpdateCategory :exec -- UPDATE name, icon, color, updated_at WHERE id AND user_id
       - DeleteCategory :exec -- DELETE WHERE id AND user_id
       - UpdateCategorySortOrder :exec -- UPDATE sort_order, updated_at WHERE id AND user_id
       - CountExpensesByCategory :one -- SELECT COUNT(*) FROM expenses WHERE category_id = $1 (this query will fail until expenses table exists in Phase 4, but define it now for API plumbing; the handler will catch the error and treat it as 0 expenses)
       NOTE: Actually, the expenses table does not exist yet. Instead of referencing it, skip CountExpensesByCategory for now. The delete endpoint should accept reassign_to param but in Phase 3, since no expenses exist, it just deletes directly. Phase 4 adds the count check.

    3. Run `cd server && sqlc generate` to generate Go code.

    4. Create `server/internal/handler/category.go`:
