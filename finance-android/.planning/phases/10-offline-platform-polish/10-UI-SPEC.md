# Phase 10 — UI Design Contract

> Visual and interaction contract for offline expense creation, sync indicators, and connectivity feedback (PLAT-02). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3) |
| Preset | not applicable |
| Component library | Material 3 (flutter/material.dart) |
| Icon library | Material Icons (built-in) |
| Font | System default (Roboto on Android, SF Pro on iOS) |

**Established project conventions (carried forward from Phases 1-9):**
- Material 3 with `Colors.teal` seed color via `ColorScheme.fromSeed`
- Zero elevation throughout (flat design) -- AppBar, Card, Dialog, BottomSheet all at elevation 0
- `FilledButton` for primary actions, `TextButton` for secondary/cancel actions
- `SnackBar` for transient feedback messages
- `ListView` with `ListTile` for data lists
- `Scaffold` + `AppBar` for all non-shell screens
- `AppScaffold` with 3-tab `NavigationBar` (History, Charts, Settings) + FAB for shell screens

## Typography

Uses Material 3 `TextTheme` from `Theme.of(context).textTheme` -- no custom font sizes. Limited to 3 sizes for this phase.

| Role | TextTheme Slot | Size | Weight | Line Height |
|------|---------------|------|--------|-------------|
| Body | bodyLarge | 16px | 400 (regular) | 1.5 (24px) |
| Label | bodySmall | 12px | 400 (regular) | 1.33 (16px) |
| Heading | titleMedium | 16px | 500 (medium) | 1.5 (24px) |

**Phase-specific usage:**
- `bodyLarge` (16px): Offline banner message text, sync status SnackBar text
- `bodySmall` (12px): "Pending sync" label beneath expense amount on unsynced tiles, sync count in banner ("3 expenses pending")
- `titleMedium` (16px, w500): Not directly added but inherited -- expense amount rendering on tiles (unchanged from existing pattern)

## Color

Uses Material 3 `ColorScheme.fromSeed(seedColor: Colors.teal)` -- all colors derived from theme, not hardcoded hex values.

| Role | Theme Token | Usage |
|------|-------------|-------|
| Dominant (60%) | `colorScheme.surface` | Scaffold background, screen backgrounds (unchanged) |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | Offline banner background |
| Accent (10%) | `colorScheme.primary` | Sync success SnackBar icon, CircularProgressIndicator during sync |
| Warning | `Colors.orange.shade700` | Offline banner leading icon, pending sync dot on expense tiles |
| Muted | `colorScheme.onSurfaceVariant` | "Pending sync" label text, offline banner body text |

**Accent reserved for:**
- CircularProgressIndicator shown during active sync
- Sync success SnackBar checkmark icon

**Warning color reserved for:**
- Offline banner leading icon (`Icons.cloud_off`)
- Small 8px pending-sync dot indicator on unsynced expense tiles

**No destructive actions in this phase.** Offline sync is automatic. Users do not manually trigger sync or delete pending items.

## Component Inventory

### Component: Offline Banner (offline_banner.dart)

**Location:** Inserted into `AppScaffold` body, above the tab content. Visible on all shell screens (History, Charts, Settings) when offline.

**States:**

1. **Online** -- Banner hidden (`SizedBox.shrink()`), no space consumed
2. **Offline, no pending** -- Banner shown with connectivity message only
3. **Offline, pending expenses** -- Banner shown with connectivity message and pending count

**Layout:**
```
Consumer:
  if connectivity == online: SizedBox.shrink()
  else:
    MaterialBanner:
      backgroundColor: colorScheme.surfaceContainerHighest
      leading: Icon(Icons.cloud_off, color: Colors.orange.shade700, size: 24)
      content: Column(crossAxisAlignment: start):
        Text("You're offline") -- bodyLarge
        if pendingCount > 0:
          Text("N expenses will sync when connected") -- bodySmall, onSurfaceVariant
      actions: [SizedBox.shrink()]  // MaterialBanner requires actions, empty
      padding: EdgeInsets.symmetric(horizontal: 16, vertical: 8)
```

**Integration point:** `AppScaffold.build` wraps the existing `child` in a `Column` with the offline banner above:
```
Column:
  OfflineBanner()       // collapses to SizedBox.shrink when online
  Expanded(child: child) // existing tab content
```

### Component: Pending Sync Indicator on Expense Tiles

**Location:** Existing `_ExpenseTile` in HistoryScreen. Modified to show a visual indicator when `expense.synced == false`.

**Layout change to existing tile:**
```
ListTile (existing):
  leading: existing 36x36 category chip (unchanged)
  title: Row:
    Text(amount) -- titleMedium (unchanged)
    if !expense.synced:
      SizedBox(width: 8)
      Container(8x8 circle, color: Colors.orange.shade700)
  subtitle: existing note text (unchanged)
  trailing: existing date text (unchanged)
```

The 8px orange dot sits inline next to the amount. This is the only visual change to the expense tile. No tooltip, no text label -- the dot alone signals "not yet synced." Once synced, the dot disappears.

### Component: Sync Status SnackBar

**Trigger:** When connectivity is restored and sync completes, show a transient SnackBar.

**Layout:**
```
SnackBar:
  content: Row:
    Icon(Icons.check_circle, color: colorScheme.primary, size: 20)
    SizedBox(width: 8)
    Text("N expenses synced") -- bodyLarge
  duration: 3 seconds
  behavior: SnackBarBehavior.floating
```

**Error variant:** If sync fails after connectivity is restored (e.g., server error):
```
SnackBar:
  content: Row:
    Icon(Icons.error_outline, color: Colors.orange.shade700, size: 20)
    SizedBox(width: 8)
    Text("Some expenses couldn't sync. Will retry automatically.") -- bodyLarge
  duration: 4 seconds
  behavior: SnackBarBehavior.floating
```

### Non-Visual: Expense Form Behavior While Offline

The existing expense creation form (`ExpenseFormScreen`) requires no visual changes. The form works identically online and offline because the local-first write pattern is transparent to the UI layer. The only observable difference is that the newly created expense appears in the History list with an orange pending-sync dot.

Category picker: Categories are served from a local sqflite cache. The picker dropdown/bottom sheet works offline using cached data. If no categories are cached (first launch while offline), the form shows the existing empty category state.

## Interaction Contracts

### Offline Banner Visibility
- Banner appears/disappears reactively via `connectivityProvider` StreamProvider
- No animation on show/hide -- MaterialBanner handles its own transitions
- Banner is visible on all three shell tabs (History, Charts, Settings) simultaneously since it lives in `AppScaffold`
- Banner is NOT shown on non-shell screens (expense form, categories, family screens) -- those screens have their own Scaffold

### Expense Creation While Offline
- User taps FAB, fills form, taps save -- identical flow to online
- Expense appears immediately in History list with orange pending-sync dot
- No error dialog, no "you are offline" warning on save -- the save always succeeds locally
- If category cache is empty (edge case: first launch offline), the category picker shows existing empty state

### Sync Trigger
- Automatic on connectivity restore (via `ref.listen` on `connectivityProvider`)
- No manual "sync now" button
- Sync processes pending queue in FIFO order (oldest first)
- If sync is already running, subsequent connectivity events are no-ops (mutex guard)

### Sync Feedback
- During sync: No visible spinner or progress indicator (sync is background, non-blocking)
- After sync success: SnackBar with count of synced expenses (3-second duration)
- After sync partial failure: SnackBar with error message (4-second duration), will retry on next connectivity event
- Pending-sync dots disappear from individual expense tiles as each is synced

### Platform Polish (Cross-Platform Consistency)
- All components use Material 3 theme tokens, not platform-specific values
- sqflite uses `sqflite_common_ffi_web` on web platform (no visual difference)
- Offline banner and sync indicators render identically on Android, iOS, and web
- No platform-conditional UI branches in this phase
