# Phase 8 — UI Design Contract

> Visual and interaction contract for family group creation, invitation, and membership management screens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3) |
| Preset | not applicable |
| Component library | Material 3 (flutter/material.dart) |
| Icon library | Material Icons (built-in) |
| Font | System default (Roboto on Android, SF Pro on iOS) |

**Established project conventions (from existing codebase):**
- Material 3 with `Colors.teal` seed color via `ColorScheme.fromSeed`
- Zero elevation throughout (flat design) -- AppBar, Card, Dialog, BottomSheet, FAB, NavigationBar all at elevation 0
- `FilledButton` for primary actions, `TextButton` for secondary/cancel actions
- `AlertDialog` for destructive confirmations
- `SnackBar` for transient feedback messages
- `ListView` with `ListTile` for settings-style lists
- `Scaffold` + `AppBar` for all non-shell screens (no bottom nav)

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding |
| sm | 8px | Between related elements (e.g. SnackBar text padding, button spacing) |
| md | 16px | Default screen padding, ListView top padding, ListTile internal spacing |
| lg | 24px | Card internal padding, section spacing between form fields |
| xl | 32px | Before primary action button in forms |
| 2xl | 48px | Major section breaks, empty state vertical centering margin |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets use Material 3 default 48px minimum (ListTile height, icon buttons).

---

## Typography

Uses Material 3 `TextTheme` from `Theme.of(context).textTheme` -- no custom font sizes.

| Role | TextTheme Slot | Size | Weight | Line Height |
|------|---------------|------|--------|-------------|
| Body | bodyMedium | 14px | 400 (regular) | 1.43 (20px) |
| Body Large | bodyLarge | 16px | 400 (regular) | 1.5 (24px) |
| Label | labelLarge | 14px | 500 (medium) | 1.43 (20px) |
| Heading | titleLarge | 22px | 400 (regular) | 1.27 (28px) |

**Phase-specific usage:**
- `titleLarge` (22px): AppBar titles ("Family", "Create Family", "Join Family"), empty state heading
- `bodyLarge` (16px): ListTile titles (member names, group name), form labels
- `bodyMedium` (14px): ListTile subtitles (email addresses, invitation status, timestamps), empty state body copy
- `labelLarge` (14px): Button text ("Create Family", "Copy Invite Link", "Leave Family")

---

## Color

Uses Material 3 `ColorScheme.fromSeed(seedColor: Colors.teal)` -- all colors derived from theme, not hardcoded hex values.

| Role | Theme Token | Usage |
|------|-------------|-------|
| Dominant (60%) | `colorScheme.surface` | Scaffold background, screen backgrounds |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | Card backgrounds (member list card, invite info card) |
| Accent (10%) | `colorScheme.primary` | Primary CTA buttons, invite link text, admin badge |
| Destructive | `colorScheme.error` | Leave Family button text, Remove Member button icon, revoke invitation |

**Accent reserved for:**
- "Create Family" FilledButton
- "Copy Invite Link" FilledButton
- "Join Family" FilledButton on accept-invite screen
- Admin role badge text
- CircularProgressIndicator

**Destructive reserved for:**
- "Leave Family" TextButton text color
- "Remove" icon button on member rows (admin view)
- "Revoke" action on pending invitation rows (admin view)
- "Delete Family" option if admin

### Screen: FamilyScreen (family_screen.dart)

1. **Loading** -- `CircularProgressIndicator` centered (matches CategoriesScreen pattern)
2. **Error** -- Centered `Text(errorMessage)` (matches CategoriesScreen pattern)

4. **FamilyLoaded (admin view)** -- Group name header, member list with remove buttons, pending invitations section, invite button

## Interaction Contracts

### Settings Integration
- New `ListTile` in `SettingsScreen` between "Categories" and "Log Out"
- Leading icon: `Icons.group`
- Title: "Family"
- Navigates to `/settings/family`

### Navigation
- `/settings/family` -- FamilyScreen (outside ShellRoute, own AppBar, no bottom nav)
- `/settings/family/create` -- CreateFamilyScreen (outside ShellRoute)
- `/invite/:token` -- AcceptInviteScreen (outside ShellRoute, deep link target)

### State Reload
- FamilyScreen calls `loadFamily()` on mount (every visit) to detect membership changes
- After create/join/leave actions, navigate back to Settings (family screen is no longer relevant after leave)
- After creating family, navigate to FamilyScreen to see new group

### Invite Flow
1. Admin taps "Copy Invite Link" on FamilyScreen
2. App calls server to create invitation, receives raw token
3. App copies `financetracker://invite/{token}` to clipboard via `Clipboard.setData()`
4. SnackBar confirms "Invite link copied"
5. Recipient opens link (deep link) or pastes code manually
6. AcceptInviteScreen shows preview, user taps "Join Family"
7. On success, navigate to FamilyScreen showing new membership

### Admin vs Member UI Differences
| Element | Admin | Member |
|---------|-------|--------|
| Remove member buttons | Visible (trailing IconButton on each non-self member) | Hidden |
| Pending Invitations section | Visible with revoke buttons | Hidden |
| "Copy Invite Link" button | Visible | Hidden |
| Leave action label | "Leave Family" (triggers delete/dissolve dialog) | "Leave Family" (triggers leave dialog) |
