1. Create `server/internal/db/queries/family_expenses.sql` with three sqlc queries:

```sql
-- name: GetFamilyExpenses :many
SELECT
    e.id,
    e.user_id,
    u.email AS user_email,
    e.category_id,
    c.name AS category_name,
    c.color AS category_color,
    c.icon AS category_icon,
    e.amount_cents,
    e.note,
    e.expense_date,
    e.created_at
FROM expenses e
JOIN family_members fm ON fm.user_id = e.user_id
JOIN users u ON u.id = e.user_id
JOIN categories c ON c.id = e.category_id
WHERE fm.family_id = $1
ORDER BY e.expense_date DESC, e.created_at DESC
LIMIT $2 OFFSET $3;

-- name: GetFamilyMemberTotals :many
SELECT
    e.user_id,
    u.email AS user_email,
    SUM(e.amount_cents)::BIGINT AS total_cents,
    COUNT(*)::INT AS expense_count
FROM expenses e
JOIN family_members fm ON fm.user_id = e.user_id
JOIN users u ON u.id = e.user_id
WHERE fm.family_id = $1
  AND e.expense_date >= $2
  AND e.expense_date <= $3
GROUP BY e.user_id, u.email
ORDER BY total_cents DESC;

-- name: GetFamilyCategoryTotals :many
SELECT
    e.category_id,
    c.name AS category_name,
    c.color AS category_color,
    c.icon AS category_icon,
    SUM(e.amount_cents)::BIGINT AS total_cents,
    COUNT(*)::INT AS expense_count
FROM expenses e
JOIN family_members fm ON fm.user_id = e.user_id
JOIN categories c ON c.id = e.category_id
WHERE fm.family_id = $1
  AND e.expense_date >= $2
  AND e.expense_date <= $3
GROUP BY e.category_id, c.name, c.color, c.icon
ORDER BY total_cents DESC;
```

- Interface: `FamilyViewDB` with methods `GetFamilyExpenses(familyID string, limit, offset int) ([]FamilyExpense, error)`, `GetFamilyMemberTotals(familyID string, dateFrom, dateTo time.Time) ([]FamilyMemberTotal, error)`, `GetFamilyCategoryTotals(familyID string, dateFrom, dateTo time.Time) ([]FamilyCategoryTotal, error)`
- Struct: `FamilyViewHandler` with fields `familyDB FamilyDB` and `viewDB FamilyViewDB`
- Constructor: `NewFamilyViewHandler(familyDB FamilyDB, viewDB FamilyViewDB) *FamilyViewHandler`
- Method `FamilyFeed(c *gin.Context)`:
  - Gets user_id from `c.GetString("user_id")`
  - Calls `h.familyDB.GetFamilyByUserID(userID)` -- returns 404 `{"error": "no family"}` on ErrFamilyNotFound, 500 on other errors
  - Parses `limit` query param (default 50, max 100) and `offset` query param (default 0) using strconv.Atoi
  - Calls `h.viewDB.GetFamilyExpenses(family.ID, limit, offset)`
  - Returns 200 with JSON array of expense objects. Each object has keys: `id`, `user_id`, `user_email`, `category_id`, `category_name`, `category_color`, `category_icon`, `amount_cents`, `note`, `expense_date` (formatted as "2006-01-02")
  - Returns empty array `[]` (not null) when no expenses
- Method `FamilySummary(c *gin.Context)`:
  - Gets user_id, resolves family via GetFamilyByUserID (same 404/500 pattern)
  - Parses `month` query param (format "2006-01"), returns 400 `{"error": "month query parameter is required (format: YYYY-MM)"}` if missing or invalid (same error message as SummaryHandler)
  - Computes dateFrom = parsed month, dateTo = last day of month (same logic as SummaryHandler: `time.Date(parsed.Year(), parsed.Month()+1, 0, 0, 0, 0, 0, time.UTC)`)
  - Calls GetFamilyMemberTotals and GetFamilyCategoryTotals
  - Computes total_cents by summing all member totals
  - Returns 200 JSON: `{"total_cents": N, "by_person": [...], "by_category": [...]}`
  - by_person items: `{"user_id": "...", "user_email": "...", "total_cents": N, "expense_count": N}`
  - by_category items: `{"category_id": "...", "category_name": "...", "category_color": "...", "category_icon": "...", "total_cents": N, "expense_count": N}`
  - Returns 0 total and empty arrays (not null) when no data
- Helper `parsePagination(c *gin.Context) (limit, offset int)` -- parses limit (default 50, clamped to 1-100) and offset (default 0, min 0) from query params
