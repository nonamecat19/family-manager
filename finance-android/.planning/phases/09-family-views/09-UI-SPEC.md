# Phase 9 — UI Design Contract

> Visual and interaction contract for the family combined expense feed (FAM-03) and family summary dashboard (FAM-04). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3) |
| Preset | not applicable |
| Component library | Material 3 (flutter/material.dart) |
| Icon library | Material Icons (built-in) |
| Font | System default (Roboto on Android, SF Pro on iOS) |

**Established project conventions (carried forward from Phase 8):**
- Material 3 with `Colors.teal` seed color via `ColorScheme.fromSeed`
- Zero elevation throughout (flat design) -- AppBar, Card, Dialog, BottomSheet all at elevation 0
- `FilledButton` for primary actions, `TextButton` for secondary/cancel actions
- `SnackBar` for transient feedback messages
- `ListView` with `ListTile` for data lists
- `Scaffold` + `AppBar` for all non-shell screens (no bottom nav)
- Month navigation pattern: chevron_left / chevron_right `IconButton` pair with `DateFormat.yMMMM()` label (from ChartsScreen)

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding |
| sm | 8px | Between related elements (e.g. subtitle spacing, chip gaps) |
| md | 16px | Default screen padding, ListView top padding, ListTile internal spacing |
| lg | 24px | Section spacing between feed and summary sections, card internal padding |
| xl | 32px | Horizontal padding around empty state content |
| 2xl | 48px | Major section breaks, empty state vertical centering margin |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets use Material 3 default 48px minimum (ListTile height, icon buttons). Month navigation chevrons are 24px icon size inside 48px touch target (matching ChartsScreen).

## Typography

Uses Material 3 `TextTheme` from `Theme.of(context).textTheme` -- no custom font sizes. Limited to 4 distinct sizes.

| Role | TextTheme Slot | Size | Weight | Line Height |
|------|---------------|------|--------|-------------|
| Body | bodyLarge | 16px | 400 (regular) | 1.5 (24px) |
| Label / Count | bodySmall | 12px | 400 (regular) | 1.33 (16px) |
| Heading | titleLarge | 22px | 400 (regular) | 1.27 (28px) |
| Display | headlineMedium | 28px | 400 (regular) | 1.29 (36px) |

**Auxiliary weight:** `titleMedium` (16px, weight 500) is used for section headers and amounts. This shares the 16px size with `bodyLarge` (differing only in weight), so it does not count as a fifth size.

**Phase-specific usage:**
- `titleLarge` (22px): AppBar titles ("Family Expenses", "Family Summary")
- `headlineMedium` (28px): Family total spent amount on summary dashboard (mirrors MonthlySummary pattern)
- `titleMedium` (16px, w500): Expense amount in feed tiles, section headers ("By Person", "By Category"), month selector label
- `bodyLarge` (16px): ListTile titles (member email in per-person breakdown), spender email subtitle on feed tiles, expense note text, "Total Spent" label, navigation ListTile subtitles
- `bodySmall` (12px): Expense count labels ("N expenses"), date trailing text on feed tiles, onSurfaceVariant color

---

## Color

Uses Material 3 `ColorScheme.fromSeed(seedColor: Colors.teal)` -- all colors derived from theme, not hardcoded hex values.

| Role | Theme Token | Usage |
|------|-------------|-------|
| Dominant (60%) | `colorScheme.surface` | Scaffold background, screen backgrounds |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | Spender email badge background (optional), dividers |
| Accent (10%) | `colorScheme.primary` | Month navigation chevrons (active state), pull-to-refresh indicator, CircularProgressIndicator |
| Muted | `colorScheme.onSurfaceVariant` | Empty state icons, expense count labels, bodySmall text |

**Accent reserved for:**
- Active month navigation chevron icons (left always, right when not current month)
- CircularProgressIndicator during loading
- Pull-to-refresh indicator color

**Category colors:** Each expense tile and category breakdown row uses the category's own stored hex color (parsed via `parseHexColor`) for the 36x36 circular category chip, matching the HistoryScreen `_CategoryChipSmall` and MonthlySummary leading widget patterns exactly.

**No destructive actions in this phase.** Family views are read-only. Edit/delete happens on the personal expense screens.

## Component Inventory

### Screen: FamilyFeedScreen (family_feed_screen.dart)

**Route:** `/settings/family/expenses` (outside ShellRoute, own AppBar, no bottom nav)

**States:**

1. **Loading** -- `CircularProgressIndicator` centered (matches HistoryScreen/CategoriesScreen pattern)
2. **Error** -- Pull-to-refresh enabled `SingleChildScrollView` with centered error text (matches ChartsScreen ChartError pattern)
3. **Empty** -- Centered icon + heading + body when no family expenses exist for the selected month
4. **Loaded** -- Month selector + scrollable list of family expense tiles with pull-to-refresh

**Layout (Loaded):**
```
Scaffold:
  AppBar: "Family Expenses"
  Column:
    // Month selector (reuse ChartsScreen pattern)
    Padding(left: 16, right: 16, top: 8, bottom: 16):
      Row(centered):
        IconButton(Icons.chevron_left, color: primary)
        Text(DateFormat.yMMMM()) -- titleMedium
        IconButton(Icons.chevron_right, color: primary or disabled)
    // Expense list
    Expanded:
      RefreshIndicator:
        ListView.builder:
          itemBuilder: _FamilyExpenseTile
```

**_FamilyExpenseTile layout:**
```
ListTile:
  leading: Container(36x36 circle, category color background)
    Icon(categoryIcon, size: 18, color: Colors.white)
  title: Text(Expense.formatCents(amountCents)) -- titleMedium
  subtitle: Column(crossAxisAlignment: start):
    Text(userEmail) -- bodyLarge, onSurfaceVariant color
    if note.isNotEmpty: Text(note) -- bodyLarge
  trailing: Column(mainAxisAlignment: center):
    Text(DateFormat.yMMMd().format(expenseDate)) -- bodySmall
```

The key difference from personal `_ExpenseTile`: the subtitle shows the spender's email as the first line (onSurfaceVariant color to distinguish from expense note), and the tile is not tappable (read-only, no edit navigation). No swipe-to-delete (users cannot delete other members' expenses).

## Interaction Contracts

### Navigation
- `/settings/family/expenses` -- FamilyFeedScreen (outside ShellRoute, own AppBar, no bottom nav)
- `/settings/family/summary` -- FamilySummaryScreen (outside ShellRoute, own AppBar, no bottom nav)
- Both accessible only from FamilyScreen (FamilyLoaded state). Not visible when user has no family.

### Month Navigation
- Both screens use the ChartsScreen month selector pattern: left chevron, `DateFormat.yMMMM()` label, right chevron
- Left chevron always active (navigate to previous month)
- Right chevron disabled (grayed via `onSurface.withAlpha(97)`) when viewing current month
- Month change triggers data reload via notifier

### Data Loading
- Both screens call their respective notifier's load method on mount (every visit) to get fresh data
- Pull-to-refresh on both screens triggers reload
- Default to current month on first visit

### State Reload on Expense Changes
- If user navigates back from family views, logs/edits/deletes a personal expense, then returns to family views, the on-mount reload fetches updated data. No cross-tab listener needed since these screens are not in the ShellRoute tab structure.

### Feed Tiles are Read-Only
- No onTap handler on `_FamilyExpenseTile` -- users cannot edit other members' expenses
- No swipe-to-delete -- family feed is view-only
- No long-press actions
